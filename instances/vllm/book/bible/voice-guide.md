# 叙述声线与风格指南（Book Bible）

> 全书统一声线的真相源。每位 writer 写作前必读。

## 叙述者人格
"白板边的内行朋友"——懂得深、讲得活。不端着，不灌输，带着读者一起读代码、推原理。

## 语言
- **大白话 + 技术深度**：先给直觉/类比，再上符号；公式后立刻给数值例子和一句人话翻译。
- 中文为主，技术术语保留英文原词（按 `glossary.json` 统一译名）。
- 句子 15–25 字，超 40 字必拆。禁书面语：综上所述 / 此处应当注意的是 / 不难看出。

## 形式度曲线（每章内）
开篇轻松（钩子）→ 原理段严谨（推导/证明）→ 小结回到轻松。每节 1–2 个轻松点，不连用。

## 正文是正式出版物（零脚手架泄漏）
- 源码引用用规范 vLLM 路径：`vllm/v1/engine/async_llm.py:L280`。**绝不**写 `instances/vllm/source/...`。
- 章节用**自然标题**。**绝不**出现 "Cell N"。
- **绝不**提及内部文件（impl-notes.md / dossier）或"详见 xxx.md / 这里截取"。
- 自包含：直接内嵌真实源码片段，删无关分支用 `# … 省略：… ` 标注；读者不开源码也能读懂。

## 伏笔与回收的呈现
- **自然融入行文**，绝不写"伏笔1 / 伏笔①"这种生硬标签（正文、图注都不要）。把它写成一句承上启下的话。
- **跨章**埋设/回收：用 markdown 链接跳到目标章，如 `[第 7 章会揭开 IPC 机制](../ch07-xxx/narrative/chapter.md)`，读者可点过去。
- **章内**回指：用 `#` 锚点链接，如 `[前面 §4.6 证过](#46-requestoutputcollector)`。
- 登记仍走 `bible.py`（due/payoff），但**登记是内部账本，呈现给读者的是自然的链接与行文**。

## 每章固定开场
Roadmap（"你在这里"）：全书地图高亮当前位置 + 上一章立了什么 / 本章解决什么 / 下一章接什么。
