# 叙述声线与风格指南（{{REPO}} Book Bible）

> 全书统一声线的真相源。每位 writer 写作前必读。
> 本指南的**通用约定承袭自既往 repo2book 实例**（尤以 vLLM 书 33 章 + v0.21.0 重基打磨）——新书**默认继承**，只按本仓特点微调（读者层级、语言、规范路径前缀）。深层"为什么"见仓库根 `wisdom/`。

## 叙述者人格
"白板边的内行朋友"——懂得深、讲得活。不端着，不灌输，带着读者一起读**真实源码**、推原理。

## 语言
- **大白话 + 技术深度**：先给直觉/类比，再上符号；公式后立刻给数值例子和一句人话翻译。
- 主语言 {{LANG_CODE}}，技术术语保留英文原词（按 `glossary.json` 统一译名）。**主线术语首现处给一次性中文注解**，如「qualname（全限定类名）」「OOT（out-of-tree）」，后文沿用英文。
- 句子 15–25 字，**超 40 字必拆**。禁书面语：综上所述 / 此处应当注意的是 / 不难看出。
- 中文句子里的标点用**全角**（，。？！；：）——`lint_punct` 校验。
- 目标读者：{{READER}}。

## 形式度曲线（每章内）
开篇轻松（钩子）→ 原理段严谨（推导/证明/数值追踪）→ 小结回到轻松。每节 1–2 个轻松点，不连用。

## 算法/机制要讲透（algorithm 维度）
非平凡机制至少给：① **2+ 轮数值追踪**——把每拍的状态变化摆成 mini 表，别只给口头结论（如"懒加载只解析一次"要用两行表展示 read#1 触发/read#2 命中缓存）；② 必要时归纳证明（基例 + 归纳步）；③ 复杂度 / 显存量化。直觉在前、符号在后、每个公式后一句人话。

## 正文是正式出版物（零脚手架泄漏）
- 源码引用用**规范路径**：`{{PREFIX}}/path/file.py:L123`（姊妹篇可对照基座，二者都用规范路径）。**绝不**写 `instances/<x>/source/...`。
- 章节用**自然标题**。**绝不**出现 "Cell N"。
- **绝不**提内部文件（dossier / impl-notes.md）或"详见 xxx.md / 这里截取"——出版物里这些不存在。
- **自包含**：直接内嵌真实源码片段，**逐字保留**（含类型标注 / 完整签名，别改写一行内的内容），删无关分支用 `# … 省略：… ` 显式标注；读者不开源码也能读懂。把**非相邻**的方法拼进一个块也要加省略标注，别让读者以为它们紧挨着。

## 公式规则（NON-NEGOTIABLE，`lint_formulas` auto-REJECT）
`\text{}`→`\mathrm{}`；`\boxed{}`→markdown 粗体标题；`\tag*{}`→`$$` 外；inline `\frac`→提升为 `$$` 块；`$$` 与内容分行；inline 仅限单符号/简单式。

## 图（diagrams）
- `gen_*.py` 拼 SVG，`rsvg-convert -z 2` 转 PNG（**勿用 ImageMagick convert**，丢中文/错位）。
- 几何要干净（`lint_diagram_geometry` 校验）：文字不越画布、不相撞、不压框、不裁切；箭头端点接到框边/端点。
- **一图一个核心对比**，细节下沉正文——信息密度过高（每框 4–5 行小字）在缩印/移动端吃力。

## 伏笔与回收的呈现
- **自然融入行文**，绝不写"伏笔1 / 伏笔①"这种生硬标签（正文、图注都不要）。写成一句承上启下的话。
- **跨章**埋设/回收：用 markdown 链接跳目标章，如 `[第 N 章会揭开 X](../chNN-xxx/narrative/chapter.md)`。
- **章内**回指：用 `#` 锚点链接（`lint_anchors` 校验锚点可解析）。
- 登记走 `bible.py`（due/payoff，内部账本）；**呈现给读者的是自然的链接与行文**。

## 每章固定开场
Roadmap（"你在这里"）：全书地图高亮当前位置 + 上一章立了什么 / 本章解决什么 / 下一章接什么——**拆成 2–3 个短句，每句 ≤25 字**，绝不塞成一段 ≥40 字的长 blockquote（读者第一口要好咽）。
